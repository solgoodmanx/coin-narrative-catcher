# Agentic Commerce Module

Use this module when token narrative includes AI agents, wallets, ACP/GDP marketplaces, or agent-to-agent payments.

## Core claim to test

"This token captures value from real agentic commerce, not just social hype."

## Evidence tiers

### Tier 1 — Hard Evidence (highest weight)
- Onchain transactions tied to service purchases
- Product/CLI artifacts proving live capability
- Verifiable payment rails (USDC/stable settlement, wallet flows)

### Tier 2 — Strong Social Evidence
- Official protocol/team announcements
- Credible builder posts with reproducible details
- Third-party technical walkthroughs

### Tier 3 — Weak Social Evidence
- Generic KOL hype
- Recycled screenshots without proof trail
- Ticker-only chatter without CA/context

## Agent customer quality checks

For recurring demand quality, prefer:
- Repeat buyers (same counterparties returning)
- Diverse counterparties (not a tiny closed loop)
- Non-circular demand (not team wallets cycling volume)
- Non-repetitive request patterns (actual varied use)

## Token capture logic checks

Ask these directly:
1. How does service usage route value to token holders?
2. Is there explicit fee sharing, discount, staking, or priority utility?
3. Is value accrual mechanical or purely narrative promise?

## Agentic narrative score (0-15)

1. Money-Rail Legitimacy (0-5)
- 5: Proven rails + live usage artifacts
- 3: Claimed rails with partial evidence
- 1: Narrative only

2. Commerce Proof (0-5)
- 5: Repeat paid demand + counterpart diversity
- 3: One-off paid events
- 1: No payment proof

3. Token Capture Clarity (0-5)
- 5: Explicit and active value capture mechanics
- 3: Planned but not live
- 1: Unclear/no capture

Interpretation:
- 12-15: Strong agentic commerce thesis
- 8-11: Promising but early/fragile
- 4-7: Mostly narrative with weak proof
- 0-3: No credible commerce leg yet
